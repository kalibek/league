# О приложении

**League Manager** — это веб-приложение для организации и запуска лиг настольного тенниса. Оно предназначено для клубов, клубов отдыха и конкурентных сообществ, которые хотят справедливую и прозрачную систему соревнований.

## Для кого это

- Клубы настольного тенниса, проводящие регулярные события лиги
- Организаторы турниров, которым нужна автоматизированная расписание и таблица
- Ойыншылар, которые хотят отслеживать свой прогресс и рейтинг с течением времени

## Как это работает

Игроки организованы в **группы** (S, A1, A2, B1, B2 и так далее) для каждого события. В каждой группе проводится полный **круговой турнир** — каждый игрок встречается с каждым другим игроком. По окончании события лучшие финалисты повышаются в более высокую группу, а худшие понижаются в более низкую группу, создавая саморазвивающуюся конкурентную лестницу.

## Основные возможности

- **Автоматическое расписание группового кругового турнира**
- **Система рейтинга Глико2** — статистически строгий алгоритм, который отслеживает навыки, неопределенность и последовательность каждого игрока
- **Живой подсчет** — результаты обновляются в реальном времени через WebSocket для мгновенного просмотра
- **Обработка DNS** — отсутствующие игроки отслеживаются и учитываются при повышении и понижении
- **Разрешение тайбреков** — четырехступенчатая логика тайбрека обеспечивает справедливое размещение даже при равных очках
- **Многоязычный интерфейс** — доступно на английском, русском и казахском языках
- **Инструменты администратора** — администраторы лиги могут настраивать размер группы, правила продвижения, количество столов и многое другое

## Рейтинг

Каждый игрок имеет **рейтинг Глико2**, который обновляется после каждого события группы. Новые игроки начинают с 1500 с высокой неопределенностью; рейтинг становится более стабильным по мере участия в большем количестве событий. Полное объяснение см. на странице [Рейтинг](rating.md).

## Движение группы

Игроки переходят между группами на основе своего финального места и настроенного количества повышений/понижений. Подробности см. на странице [Движение](movements.md).
