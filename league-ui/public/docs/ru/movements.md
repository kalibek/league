# Переходы между группами

## Структура дивизионов

Игроки организованы в иерархию групп. Высшая группа — **S** (суперлига), затем **A1**, **A2**, **B1**, **B2** и так далее. Каждая группа играет отдельно внутри мероприятия, но результаты определяют движение игроков на следующий месяц.

```
S  ←→  A1  ←→  A2  ←→  B1  ←→  B2  ←→  ...
```

Повысившиеся из A1 заполняют места в S; пониженные из S переходят в A1. Цепочка продолжается по всем дивизионам.

## Повышение и понижение

Для каждой группы задаются два числа:

- **Количество повышений** — сколько лучших игроков переходят в группу выше
- **Количество понижений** — сколько худших игроков переходят в группу ниже

Эти параметры устанавливает администратор лиги при создании и могут различаться для разных групп.

## Неявка (DNS)

Игрок с отметкой **DNS** (Did Not Show) не явился на мероприятие. Такие игроки всегда понижаются, независимо от своего положения в таблице.

Игроки DNS также занимают слоты понижения по следующему правилу:

```
delta = КоличествоПонижений − количествоDNS
```

- Если **delta > 0**: нижние `delta` активных (не DNS) игроков также понижаются.
- Если **delta ≤ 0**: все слоты понижения уже заняты DNS-игроками; активные игроки не понижаются.

**Пример:** В группе количество понижений = 2, один игрок DNS.
- delta = 2 − 1 = 1
- Игрок DNS понижается, плюс 1 нижний активный игрок также понижается.

**Пример:** В группе количество понижений = 2, три игрока DNS.
- delta = 2 − 3 = −1 (≤ 0)
- Все три DNS-игрока понижаются, активные игроки не понижаются.

## Разрешение ничьих

Если двое или более игроков набрали одинаковое количество побед, применяются следующие этапы по порядку:

1. **Основные очки** — сумма очков во всех матчах группы.
2. **Очки тайбрейка** — разница геймов (выигранных минус проигранных) только в матчах между игроками, которые находятся в ничьей.
3. **Соотношение побед/поражений** — победы делятся на поражения по всем матчам в группе.
4. **Решение арбитра** — если ничья не устранена после шагов 1–3, место определяет арбитр вручную.

### Конкретный пример

Три игрока — Алиса, Борис и Карина — все заканчивают с 2 победами и 1 поражением.

- **Шаг 1:** У всех троих по 6 очков. Ничья сохраняется.
- **Шаг 2:** Среди матчей Алиса–Борис, Алиса–Карина, Борис–Карина:
  - Алиса: разница геймов +2
  - Борис: разница геймов 0
  - Карина: разница геймов −2
  - Карина выбывает из тайбрейка. Алиса занимает место выше Бориса.
- Итоговый порядок: **Алиса → Борис → Карина**.

Если бы и шаг 2 дал равный результат, проверялся бы шаг 3 (соотношение побед/поражений), затем шаг 4.
