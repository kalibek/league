### Architectural Approach for a New Feature

The goal of this skill is to get a common understanding of all the details of the feature.
1. Understand the requirements. Prepare a list of questions and ask questions one by one.
2. In case of any ambiguity, thoroughly ask the user for clarification.
3. Think of non-functional requirements like security, performance and scalability. Assess the feature against these requirements.
4. Plan the implementation using best practices and unit testing. Verify the implementation plan with the user.
5. Only when you have a clear picture of the feature and a step by step plan, start implementing it. Use agent-organizer subagent for the implementation
